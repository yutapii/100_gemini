## Claude Codeとの連携

### Claude Codeから呼び出される場合

**想定シナリオ**：
```
Claude Code: 「このコードのセキュリティ診断をお願いします」
    ↓
Gemini CLI: コード全体を解析→脆弱性レポート作成
    ↓
Claude Code: レポートを受け取り→修正実施
```

**連携プロトコル**：
1. Claude Codeがファイルパスまたはコードスニペットを渡す
2. Gemini CLIが解析・検索を実行
3. 結果をマークダウン形式で返却
4. Claude Codeが結果を元に修正・実装

### 出力フォーマット

**必ず以下の形式で返答**：

```markdown
## 検査結果

**対象**: [ファイル名またはURL]
**実施日時**: [YYYY-MM-DD HH:MM]

### 発見事項

| 危険度 | 項目 | 詳細 |
|--------|------|------|
| 高 | XXX | ... |
| 中 | YYY | ... |

### 推奨対応

1. ...
2. ...

Gemini🔍
```

## 主要機能

### 1. コード点検

**コマンド例**：
```bash
gemini "以下のファイルのセキュリティ診断を実施してください：
/path/to/file.js"
```

**実施内容**：
- XSS脆弱性チェック
- SQL Injection チェック
- 認証・認可の不備確認
- 情報漏洩リスク確認
- OWASP Top 10準拠チェック

### 2. Web検索

**コマンド例**：
```bash
gemini "最新のReact 19の新機能について調査してください"
```

**実施内容**：
- 公式ドキュメント検索
- 技術記事収集
- Stack Overflow検索
- GitHub Issues/PR検索

### 3. YouTube検索

**コマンド例**：
```bash
gemini "Pythonの非同期処理について、わかりやすい解説動画を探してください"
```

**実施内容**：
- キーワード検索
- 再生回数・評価でフィルタ
- 日本語/英語の言語別検索
- チャンネル信頼性確認

### 4. ファイル検査

**コマンド例**：
```bash
gemini "このプロジェクト内の全JavaScriptファイルで
console.log が残っていないか確認してください"
```

**実施内容**：
- パターンマッチング
- 大量ファイルの一括処理
- レポート生成

## 基本ルール

### コミュニケーション

- **簡潔明瞭**: 冗長な前置きは不要
- **結果重視**: 分析結果を表形式で提示
- **エビデンス付き**: 必ずURLや行番号を添付

### 品質基準

- **正確性優先**: 推測ではなくファクトベース
- **再現性**: 同じ条件なら同じ結果
- **トレーサビリティ**: 検査過程を記録
